About this case file

Why Incident Desk exists

Most small teams don't lack security tools — they lack a place to write down what happened. Incident Desk is a deliberately small system built around one job: take a report, give it a status, and keep that status honest until the case closes.

What it does

Any registered user can file a report describing a phishing attempt, malware, an unauthorized access attempt, or another security event. Analysts and administrators triage the queue, attach evidence, change status as work progresses, and close the case once it's resolved. Every action is written to an activity log automatically, so the history can't be edited away.

What it deliberately leaves out

This system does not detect threats, watch network traffic, or predict attacks — it is a record-keeping and workflow tool, not a SIEM. That scope is intentional: a small team's first need is almost always "where do we even write this down," and that's the gap this project closes.
